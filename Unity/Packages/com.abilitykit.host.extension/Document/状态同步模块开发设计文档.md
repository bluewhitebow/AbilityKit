## 一、设计理念：什么是状态同步？

### 1.1 状态同步 vs 帧同步

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     状态同步核心思想                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  【帧同步】：同步输入，每个客户端自己计算结果                              │
│  【状态同步】：同步状态，服务器计算结果并广播                              │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                                                                  │  │
│  │   客户端 A ──┬── 输入 ──▶ 服务器 ──┬── 广播状态 ──▶ 客户端 B   │  │
│  │              │                    │                             │  │
│  │              │                    │                             │  │
│  │              ▼                    ▼                             │  │
│  │         客户端本地              服务器                           │  │
│  │         不计算游戏逻辑         执行所有游戏逻辑                  │  │
│  │         只渲染画面             存储完整游戏状态                  │  │
│  │                                                                  │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 状态同步的优势

```
✅ 状态同步的优势：

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  1. 安全性高                                                           │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  服务器是唯一的真相来源                                           │  │
│  │  客户端无法伪造游戏逻辑                                           │  │
│  │  适合需要严格防作弊的游戏                                          │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  2. 容错性强                                                           │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  网络丢包只影响状态更新，不影响游戏运行                           │  │
│  │  客户端可以本地预测，并在收到服务器状态时纠正                      │  │
│  │  适合网络条件不稳定的场景                                        │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  3. 断线重连友好                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  服务器保存完整状态，重连后可以快速恢复                            │  │
│  │  只需要同步当前状态，不需要重演历史帧                               │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  4. 实现简单                                                           │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  客户端不需要实现完整的游戏逻辑                                    │  │
│  │  只需要接收和渲染服务器状态                                       │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.3 状态同步的挑战

```
❌ 状态同步的挑战：

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  1. 带宽消耗大                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  每帧需要传输所有实体状态                                         │  │
│  │  实体数量多时，带宽成为瓶颈                                       │  │
│  │                                                                  │  │
│  │  解决方案：                                                       │  │
│  │  - 增量同步（只传输变化的状态）                                   │  │
│  │  - 压缩序列化                                                     │  │
│  │  - 优先级策略（关键实体高频同步）                                  │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  2. 延迟感知明显                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  玩家操作后需要等待服务器确认才能看到结果                          │  │
│  │  网络延迟高时，操作手感差                                          │  │
│  │                                                                  │  │
│  │  解决方案：                                                       │  │
│  │  - 客户端本地预测                                                 │  │
│  │  - 延迟补偿                                                       │  │
│  │  - 特效先行（表现先播放，效果后验证）                              │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  3. 服务器压力大                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │  服务器需要计算所有游戏逻辑                                        │  │
│  │  实体数量多时，服务器性能成为瓶颈                                  │  │
│  │                                                                  │  │
│  │  解决方案：                                                       │  │
│  │  - 分布式服务器架构                                               │  │
│  │  - 区域化服务器                                                   │  │
│  │  - 客户端预测减少服务器 Tick 频率                                  │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 三、快照路由系统

### 3.1 快照分发器

```csharp
namespace AbilityKit.Ability.Snapshot
{
    /// <summary>
    /// 帧快照分发器
    /// 根据 OpCode 路由快照到对应的处理器
    /// </summary>
    public sealed class FrameSnapshotDispatcher
    {
        private readonly ISnapshotDecoderRegistry _decoders;
        private readonly ISnapshotHandlerRegistry _handlers;

        /// <summary>
        /// 接收快照信封并分发处理
        /// </summary>
        public void Feed(ISnapshotEnvelope envelope)
        {
            if (envelope.Snapshot is not { } snapshot)
                return;

            // 1. 根据 OpCode 获取解码器
            if (!_decoders.TryGetDecoder(snapshot.OpCode, out var decoder))
            {
                // 没有解码器，尝试直接处理原始数据
                DispatchRaw(snapshot);
                return;
            }

            // 2. 解码快照
            if (!decoder(in snapshot, out var decoded))
            {
                Log.Warning($"[Snapshot] Decode failed for OpCode {snapshot.OpCode}");
                return;
            }

            // 3. 分发到处理器
            Dispatch(decoded, envelope);
        }

        /// <summary>
        /// 分发解码后的数据
        /// </summary>
        private void Dispatch<T>(T data, ISnapshotEnvelope envelope)
        {
            // 调用所有注册的处理程序
            if (_handlers.TryGetHandlers<T>(out var handlers))
            {
                foreach (var handler in handlers)
                {
                    try
                    {
                        handler.Handle(envelope, data);
                    }
                    catch (Exception ex)
                    {
                        Log.Exception(ex, $"[Snapshot] Handler error: {handler.GetType().Name}");
                    }
                }
            }
        }
    }
}
```

### 3.2 解码器注册

```csharp
namespace AbilityKit.Ability.Snapshot
{
    /// <summary>
    /// 快照解码器注册接口
    /// </summary>
    public interface ISnapshotDecoderRegistry
    {
        /// <summary>
        /// 解码器委托
        /// </summary>
        public delegate bool TryDecode<T>(in WorldStateSnapshot snap, out T value);

        /// <summary>
        /// 注册解码器
        /// </summary>
        void RegisterDecoder<T>(int opCode, TryDecode<T> decoder);
    }

    /// <summary>
    /// 解码器注册实现
    /// </summary>
    public sealed class SnapshotDecoderRegistry : ISnapshotDecoderRegistry
    {
        private readonly Dictionary<int, Delegate> _decoders = new();

        public void RegisterDecoder<T>(int opCode, TryDecode<T> decoder)
        {
            _decoders[opCode] = decoder;
        }

        public bool TryGetDecoder<T>(int opCode, out TryDecode<T> decoder)
        {
            if (_decoders.TryGetValue(opCode, out var del) && del is TryDecode<T> typed)
            {
                decoder = typed;
                return true;
            }
            decoder = null;
            return false;
        }
    }
}
```

## 四、快照管线

### 4.1 管线阶段

```csharp
namespace AbilityKit.Ability.Snapshot
{
    /// <summary>
    /// 快照管线阶段注册接口
    /// </summary>
    public interface ISnapshotPipelineStageRegistry
    {
        /// <summary>
        /// 添加管线阶段
        /// </summary>
        /// <param name="opCode">操作码</param>
        /// <param name="order">执行顺序（从小到大）</param>
        /// <param name="handler">处理函数</param>
        /// <returns>注销令牌</returns>
        IDisposable AddPipelineStage<T>(
            int opCode,
            int order,
            Action<object, ISnapshotEnvelope, T> handler);
    }

    /// <summary>
    /// 快照管线
    /// 按顺序执行多个处理阶段
    /// </summary>
    public sealed class SnapshotPipeline
    {
        private readonly List<PipelineStage> _stages = new();

        public void AddStage<T>(int opCode, int order, Action<object, ISnapshotEnvelope, T> handler)
        {
            _stages.Add(new PipelineStage
            {
                OpCode = opCode,
                Order = order,
                Handler = handler
            });
            _stages.Sort((a, b) => a.Order.CompareTo(b.Order));
        }

        /// <summary>
        /// 执行管线
        /// </summary>
        public void Execute(object context, ISnapshotEnvelope envelope)
        {
            if (envelope.Snapshot is not { } snapshot)
                return;

            foreach (var stage in _stages)
            {
                if (stage.OpCode != snapshot.OpCode)
                    continue;

                try
                {
                    stage.Handler(context, envelope, /* 解码后的数据 */);
                }
                catch (Exception ex)
                {
                    Log.Exception(ex, $"[Pipeline] Stage error: {stage.Order}");
                }
            }
        }

        private struct PipelineStage
        {
            public int OpCode;
            public int Order;
            public Delegate Handler;
        }
    }
}
```

## 五、帧缓冲与延迟控制

### 5.1 远程帧缓冲

```csharp
namespace AbilityKit.Ability.Snapshot
{
    /// <summary>
    /// 远程帧缓冲
    /// 管理从网络接收的帧数据，支持延迟播放
    /// </summary>
    public sealed class RemoteFrameBuffer<TFrame> : IRemoteFrameSource<TFrame>, IRemoteFrameSink<TFrame>
    {
        private readonly Dictionary<int, TFrame> _byFrame = new();
        private int _minFrame = int.MaxValue;
        private int _maxReceivedFrame = -1;
        private int _maxConfirmedFrame = -1;

        /// <summary>
        /// 延迟帧数
        /// 用于网络抖动缓冲
        /// </summary>
        public int DelayFrames { get; set; }

        /// <summary>
        /// 最大接收帧号
        /// </summary>
        public int MaxReceivedFrame => _maxReceivedFrame;

        /// <summary>
        /// 目标帧号 = 最大接收帧 - 延迟帧
        /// </summary>
        public int TargetFrame => Math.Max(-1, _maxReceivedFrame - DelayFrames);

        /// <summary>
        /// 接收帧数据
        /// </summary>
        public void Receive(int frame, TFrame data)
        {
            _byFrame[frame] = data;
            _maxReceivedFrame = Math.Max(_maxReceivedFrame, frame);
        }

        /// <summary>
        /// 获取帧数据
        /// </summary>
        public bool TryGet(int frame, out TFrame data)
        {
            return _byFrame.TryGetValue(frame, out data);
        }

        /// <summary>
        /// 确认帧（已处理）
        /// </summary>
        public void Confirm(int frame)
        {
            _maxConfirmedFrame = Math.Max(_maxConfirmedFrame, frame);

            // 清理旧数据
            var threshold = Math.Min(_minFrame, _maxConfirmedFrame - 60);
            if (threshold < int.MaxValue)
            {
                var keys = _byFrame.Keys.Where(k => k < threshold).ToList();
                foreach (var key in keys)
                    _byFrame.Remove(key);
                _minFrame = _byFrame.Keys.DefaultIfEmpty(int.MaxValue).Min();
            }
        }

        /// <summary>
        /// 清理指定帧之前的数据
        /// </summary>
        public void TrimBefore(int minFrameInclusive)
        {
            var keys = _byFrame.Keys.Where(k => k < minFrameInclusive).ToList();
            foreach (var key in keys)
                _byFrame.Remove(key);
        }
    }
}
```

### 5.2 帧抖动缓冲

```csharp

{

    /// <summary>
    /// 帧抖动缓冲
    /// 处理网络抖动导致的帧丢失
    /// </summary>
    public sealed class FrameJitterBuffer<T>
    {
        private readonly int _delayFrames;
        private readonly MissingFrameMode _mode;
        private readonly T _defaultValue;
        private readonly Dictionary<int, T> _buffer = new();
        private int _lastValidFrame = -1;

        public FrameJitterBuffer(int delayFrames, MissingFrameMode mode, T defaultValue)
        {

        /// <summary>
        /// 添加帧数据
        /// </summary>
        public void Add(int frame, T data)
        {
            _buffer[frame] = data;
            _lastValidFrame = frame;
            _lastValidValue = data;
        }

        /// <summary>
        /// 获取目标帧数据（应用延迟）
        /// </summary>
        public bool TryGet(int targetFrame, out T data)
        {
            var sourceFrame = targetFrame - _delayFrames;

            if (_buffer.TryGetValue(sourceFrame, out data))
            {
                return true;
            }

            // 处理丢帧
            return _mode switch
            {
                MissingFrameMode.FillDefault => (data = _defaultValue) != null,
                MissingFrameMode.RepeatLast => (_lastValidFrame >= 0) && (data = _lastValidValue) != null,
                MissingFrameMode.Skip => false,
                _ => false
            };
        }
    }
}
```

### 5.3 帧聚合器

```csharp
namespace AbilityKit.Ability.Snapshot
{
    /// <summary>
    /// 远程帧聚合器
    /// 合并同一帧的多条消息
    /// </summary>
    public sealed class RemoteFrameAggregator
    {
        private readonly Dictionary<int, List<PlayerInputCommand>> _inputsByFrame = new();
        private readonly Dictionary<int, List<ISnapshotEnvelope>> _envelopesByFrame = new();

        /// <summary>
        /// 添加帧包
        /// </summary>
        public void AddPacket(FramePacket packet)
        {
            int frame = packet.Frame.Value;

            // 合并输入
            if (packet.Inputs != null && packet.Inputs.Count > 0)
            {
                if (!_inputsByFrame.TryGetValue(frame, out var list))
                {
                    list = new List<PlayerInputCommand>();
                    _inputsByFrame[frame] = list;
                }
                list.AddRange(packet.Inputs);
            }

            // 合并快照
            if (packet.Snapshot.HasValue)
            {
                if (!_envelopesByFrame.TryGetValue(frame, out var list))
                {
                    list = new List<ISnapshotEnvelope>();
                    _envelopesByFrame[frame] = list;
                }
                list.Add(packet);
            }
        }

        /// <summary>
        /// 构建输入帧
        /// </summary>
        public RemoteInputFrame BuildInputFrame(FrameIndex frame)
        {
            _inputsByFrame.TryGetValue(frame.Value, out var inputs);
            return new RemoteInputFrame(frame, inputs ?? Array.Empty<PlayerInputCommand>());
        }

        /// <summary>
        /// 构建快照帧
        /// </summary>
        public RemoteSnapshotFrame BuildSnapshotFrame(FrameIndex frame)
        {
            _envelopesByFrame.TryGetValue(frame.Value, out var envelopes);
            return new RemoteSnapshotFrame(frame, envelopes ?? Array.Empty<ISnapshotEnvelope>());
        }

        /// <summary>
        /// 清理已处理的帧
        /// </summary>
        public void ClearBefore(FrameIndex frame)
        {
            var threshold = frame.Value;
            foreach (var key in _inputsByFrame.Keys.Where(k => k < threshold).ToList())
                _inputsByFrame.Remove(key);
            foreach (var key in _envelopesByFrame.Keys.Where(k => k < threshold).ToList())
                _envelopesByFrame.Remove(key);
        }
    }
}
```

```csharp
namespace AbilityKit.Ability.Snapshot
{
    /// <summary>
    /// 帧包网络适配器上下文
    /// </summary>
    public interface IFramePacketNetAdapterContext
    {
        /// <summary>
        /// 输入延迟帧数
        /// </summary>
        int InputDelayFrames { get; }

        /// <summary>
        /// 远程驱动的世界（权威状态）
        /// </summary>
        IWorld RemoteDrivenWorld { get; }

        /// <summary>
        /// 已确认的世界（本地状态）
        /// </summary>
        IWorld ConfirmedWorld { get; }

        /// <summary>
        /// 快照分发器
        /// </summary>
        FrameSnapshotDispatcher Snapshots { get; }
    }

    /// <summary>
    /// 帧包网络适配器
    /// 处理从网络接收的帧包
    /// </summary>
    public sealed class FramePacketNetAdapter
    {
        private readonly IFramePacketNetAdapterContext _ctx;
        private readonly RemoteFrameBuffer<FramePacket> _frameBuffer;
        private readonly FrameJitterBuffer<PlayerInputCommand[]> _inputJitterBuffer;

        public FramePacketNetAdapter(IFramePacketNetAdapterContext ctx)
        {
            _ctx = ctx;
            _frameBuffer = new RemoteFrameBuffer<FramePacket>();
            _inputJitterBuffer = new FrameJitterBuffer<PlayerInputCommand[]>(
                ctx.InputDelayFrames,
                MissingFrameMode.RepeatLast,
                Array.Empty<PlayerInputCommand>());
        }

        /// <summary>
        /// 处理帧包并注入到系统
        /// </summary>
        public FramePacket ProcessAndFeed(FramePacket packet)
        {
            // 1. 放入缓冲
            _frameBuffer.Receive(packet.Frame.Value, packet);

            // 2. 处理输入（如果有）
            packet = ProcessInput(packet);

            // 3. 注入快照到分发器
            _ctx.Snapshots?.Feed(packet);

            return packet;
        }

        /// <summary>
        /// 处理输入数据
        /// </summary>
        private FramePacket ProcessInput(FramePacket packet)
        {
            if (packet.Inputs == null || packet.Inputs.Count == 0)
                return packet;

            // 添加到抖动缓冲
            foreach (var input in packet.Inputs)
            {
                // 延迟处理输入
                _inputJitterBuffer.Add(input.Frame.Value, /* 收集 */);
            }

            return packet;
        }
    }
}
```

### 8.2 设计选择

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| MOBA、格斗游戏 | 帧同步 | 需要高一致性、低延迟手感 |
| MMORPG、大型多人 | 状态同步 | 玩家多、安全性要求高 |
| RTS、策略游戏 | 帧同步 | 确定性要求高、支持录像 |
| 休闲游戏 | 状态同步 | 实现简单、容错率高 |
| FPS、射击游戏 | 混合方案 | 移动预测、射击命中服务器校验 |

## 九、设计总结

### 9.1 核心优势

| 特性 | 说明 |
|------|------|
| **通用性** | `OpCode + Payload` 架构支持任意类型快照 |
| **可扩展** | 通过注册机制轻松添加新快照类型 |
| **模块化** | 解码、路由、处理分离 |
| **容错性** | 帧缓冲和抖动缓冲处理网络不稳定 |
| **可追踪** | 管线阶段支持日志和调试 |

### 9.2 扩展点

1. **自定义编码器**：实现 `TryDecode<T>` 委托
2. **自定义管线阶段**：通过 `ISnapshotPipelineStageRegistry` 添加
3. **自定义缓冲策略**：扩展 `FrameJitterBuffer` 或 `RemoteFrameBuffer`
4. **多世界支持**：每个世界独立的快照路由上下文

### 9.3 注意事项

- 快照 Payload 应尽量精简，减少带宽消耗
- 解码器应处理异常，避免单条快照失败影响整体
- 帧缓冲容量需要根据网络延迟合理设置
- 对于关键快照（如伤害事件），应考虑可靠性传输
